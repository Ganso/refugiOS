# Contributing to refugiOS / Contribuyendo a refugiOS

[English](#english) | [Español](#español)

---

<a name="english"></a>
## English

Welcome to **refugiOS**! We are glad you're here. This project is dedicated to digital resilience and autonomy, and your help is vital to make it more robust.

### How to contribute
- **Report Bugs:** If you find a bug, please create an Issue and include steps to reproduce it.
- **Feature Requests:** Have a new idea? Open an Issue to discuss it before starting development.
- **Documentation:** Help us improve our docs or translate them.
- **Code:** You are welcome to submit Pull Requests (PRs).

### Guidelines
1. **Fork the repo.**
2. **Create a branch:** `git checkout -b feature/your-feature-name`.
3. **Commit messages:** Please write them in **English**.
4. **Pull Request:** Link your PR to the corresponding Issue.
5. **Code Language:** Please keep all code comments, variable names, and logic in **English** for international consistency.

## Español

¡Bienvenido/a a **refugiOS**! Nos alegra que estés aquí. Este proyecto está dedicado a la resiliencia y autonomía digital, y tu ayuda es vital para hacerlo más robusto.

### Cómo contribuir
- **Reportar errores:** Si encuentras un fallo, crea un Issue e incluye los pasos para reproducirlo.
- **Sugerencias:** ¿Tienes una idea nueva? Abre un Issue para discutirla antes de empezar a desarrollar.
- **Documentación:** Ayúdanos a mejorar nuestra documentación o a traducirla.
- **Código:** Eres bienvenido/a a enviar Pull Requests (PRs).

### Directrices
1. **Haz un Fork del repositorio.**
2. **Crea una rama:** `git checkout -b feature/nombre-de-tu-mejora`.
3. **Mensajes de commit:** Por favor, escríbelos en **inglés**.
4. **Pull Request:** Vincula tu PR al Issue correspondiente.
5. **Idioma del código:** Por favor, mantén los comentarios del código, nombres de variables y lógica en **inglés** para mantener la consistencia internacional.
